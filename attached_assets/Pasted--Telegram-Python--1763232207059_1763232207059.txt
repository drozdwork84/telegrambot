ТЗ:

Нужно написать Telegram-бота на Python для напоминаний о предстоящих матчах.

Требования к стеку и запуску





Язык: Python 3.



Библиотека для Telegram: желательно aiogram (асинхронная), если сложно — можно python-telegram-bot. Главное — стабильный постоянный поллинг.



Хранилище: SQLite (файл в корне проекта, например matches.db).



Конфигурация:





Telegram токен берётся из переменной окружения BOT_TOKEN.



Часовой пояс — Московское время (Europe/Moscow). Использовать таймзону через zoneinfo или pytz, чтобы время было именно московским.



Приложение должно корректно работать на Replit как постоянный процесс (основной main.py запускает бота и планировщик напоминаний).

Функционал бота





Бот принимает любой текст (в том числе очень длинный, до лимита сообщений Telegram) и пытается вычленить из него список матчей.



Каждый матч должен иметь как минимум:





дату и время начала (в московском времени);



название/описание матча (строка, которую потом покажем в напоминании).



За 1 минуту до начала каждого матча бот отправляет в соответствующий чат сообщение:





формат: Через минуту матч в HH:MM — <название матча>.



Время считать московским, вне зависимости от локального сервера.



Бот работает раздельно по чатам:





матчи, присланные из одного чата, сохраняются и напоминаются только в этот же чат;



у каждого чата — свой список матчей.

Примеры входных данных
Пользователь может прислать многострочный текст вроде (не изменять пример, взять его как есть):

16.11.2025 08:55	110	Дроздовский Юрий	Динамо-Ульяновск (ж) — Старт Йошкар-Ола (ж)	Россия. Женщины. 1-я лига	Волейбол	-	SportLevel (STDM)	Russia	SportLevel (STDM), Видеотрансляции: ФонТВ-8 (SportLevel)
16.11.2025 09:55	110	Дроздовский Юрий	Анкара Баросу — Токат Блд.	Турция. 2-я лига	Волейбол	-	BetRadar (STDM)	Turkiye	BetRadar (STDM)
16.11.2025 09:55	110	Дроздовский Юрий	Ханкук (ж) — Тэджон (ж)	Юж. Корея. Женщины. V-Лига	Волейбол Топ	-	SportLevel (TV)	Russia	SportLevel (TV), BetRadar (TV), BetGenius (TV)
16.11.2025 10:55	110	Дроздовский Юрий	Институт Монтсеррат-Сант Герваси U19 (ж) — Терраса А U19 (ж)	Испания. Женщины. До 19 лет. Региональная лига	Волейбол	-	SportLevel (STDM)	Spain	SportLevel (STDM)


Из такого текста бот должен:





распознать 4 матча;



для каждого матча записать дату/время:





16.11.2025 08:55



16.11.2025 09:55



16.11.2025 09:55



16.11.2025 10:55



в качестве названия матча взять, например, 4-е текстовое поле (после даты/времени и двух первых колонок), то есть:





Динамо-Ульяновск (ж) — Старт Йошкар-Ола (ж)



Анкара Баросу — Токат Блд.



Ханкук (ж) — Тэджон (ж)



Институт Монтсеррат-Сант Герваси U19 (ж) — Терраса А U19 (ж)

Важно: входной формат может отличаться. Нужно написать устойчивый парсер, который:





Разбивает входной текст по строкам.



Для каждой строки:





пытается найти дату и время в начале строки;



сначала пробует формат DD.MM.YYYY HH:MM;



дополнительно можно попробовать ещё несколько распространённых форматов, например:





DD.MM.YY HH:MM



YYYY-MM-DD HH:MM



DD/MM/YYYY HH:MM



всё, что идёт после даты и времени, считать «хвостом» строки, в котором есть название матча.



Название матча:





взять как «самое содержательное текстовое поле» после даты/времени;



для таб-разделённых строк — можно считать, что это 4-я колонка (как в примере), но нужно предусмотреть, что колонок может быть меньше/больше.



если автоматически определить сложно — можно просто взять текст после первых двух–трёх «служебных» полей в середине строки, главное — чтобы в напоминании было понятно, какой матч.

Если строку распарсить не удалось (нет распознаваемой даты/времени) — её можно игнорировать или логировать как ошибку, но не должно быть падения бота.

Команды бота
Нужно реализовать минимум следующие команды:





/start





Приветственное сообщение и краткая инструкция:





что бот принимает большие простыни текста с матчами;



что он будет напоминать за 1 минуту до начала каждого матча;



что есть команда для просмотра матчей на сегодня.



/today





Показывает все будущие матчи на сегодня (по московскому времени) в этом чате.



Формат ответа:





один матч на строку: HH:MM — <название матча>;



матчи отсортированы по времени.



Если на сегодня нет будущих матчей — выводит понятное сообщение («На сегодня будущих матчей нет»).

Дополнительно (по желанию, если несложно):
3. /all или /next





Список всех будущих матчей (не только на сегодня), но это не обязательно, главное — /today.

Обработка обычных сообщений





Любое текстовое сообщение, которое не является командой, бот воспринимает как список матчей (может быть одна строка, может быть 50 строк).



Бот:





парсит текст;



для каждой успешно распознанной строки создаёт запись матча в базе данных;



привязывает матч к:





chat_id (и при необходимости к user_id, но достаточно chat_id);



отвечает кратким итогом, например:





Добавлено матчей: 4 (если удалось распарсить 4 строки).



Если ни одной строки распарсить не удалось, бот пишет об этом, чтобы пользователь понял, что данные не принялись.

Хранение данных (SQLite)
Нужна таблица, например matches со столбцами:





id INTEGER PRIMARY KEY AUTOINCREMENT,



chat_id INTEGER (или TEXT),



match_datetime (дата/время в таймзоне Europe/Moscow, можно хранить как ISO-строку),



title TEXT (название матча),



reminded BOOLEAN/INTEGER (0/1, флаг, отправлено ли напоминание),



created_at (дата/время создания записи).

Логика напоминаний





Нужен фоновый планировщик (асинхронная задача или отдельный поток), который регулярно проверяет базу данных.



Период проверки — примерно раз в 30 секунд (можно чаще/реже, но без сильной нагрузки).



На каждой итерации:





берём текущее московское время now_msk;



ищем все матчи, для которых:





reminded = 0;



match_datetime - 1 минута <= now_msk < match_datetime (то есть «примерно за минуту до начала»);



для каждого такого матча:





отправляем сообщение в соответствующий chat_id:





Через минуту матч в HH:MM — <название матча>;



выставляем reminded = 1, чтобы не дублировать напоминания.



Нужна защита от повторных напоминаний при перезапуске бота:





если бот был выключен и включился позже, чем «минус 1 минута до матча», можно:





либо вообще не напоминать уже просроченные матчи;



либо добавить простое правило: напоминать только если match_datetime > now_msk.

Работа с часовыми поясами





Чётко использовать часовой пояс Europe/Moscow:





при парсинге входной даты/времени — считать, что всё, что прислал пользователь, уже в московском времени;



хранить в базе либо в UTC с привязкой, либо сразу в Московском, но консистентно во всём коде.



Все сравнения времени (в том числе в планировщике) делать в одном и том же часовом поясе.

Структура проекта





main.py — точка входа:





инициализация бота и диспетчера;



подключение к базе;



запуск обработчиков команд и сообщений;



запуск фонового планировщика напоминаний.



Модуль/функции для:





парсинга текста в список матчей;



работы с базой данных (создание таблиц, добавление матчей, выборка будущих матчей на сегодня, выборка матчей для напоминаний).



Код должен быть относительно читаемым, без лишней магии, с комментариями к ключевым местам (парсинг, напоминания, работа с таймзоной).

Проверка





Нужен простой сценарий проверки:





отправить боту примерный текст с несколькими матчами;



убедиться, что:





бот отвечает, сколько матчей добавлено;



команда /today показывает их корректно;



если поставить матчи на ближайшие минуты, бот пришлёт напоминание за минуту до начала.



Весь код должен быть в одном проекте, готовом к запуску на Replit.
